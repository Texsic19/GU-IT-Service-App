# 🎫 Gonzaga University IT Help Desk Ticketing System

A full-stack IT ticketing system built for Gonzaga University's IT Services department. Built with **Streamlit**, **PostgreSQL (Supabase)**, and **Google Gemini AI**.

**Live App:** [your-app-url.streamlit.app](https://your-app-url.streamlit.app)

---

## What This System Does

This system allows GU students, staff, and faculty to submit IT support tickets, which are automatically categorized and prioritized using AI. IT technicians can manage tickets, assign them, update their status, and receive AI-generated step-by-step fix suggestions — similar to how Nessus provides remediation guidance for security vulnerabilities.

---

## Features

- **AI Auto-Categorization** — Google Gemini automatically assigns category and priority on ticket submission
- **AI Fix Suggestions** — Each ticket gets a step-by-step resolution guide generated by Gemini
- **Full CRUD** — Create, read, update, and delete tickets, users, and technicians
- **Search & Filter** — Filter tickets by status, priority, category, or keyword
- **Tag System** — Many-to-many tagging of tickets
- **Dashboard** — Live metrics and charts by category and priority

## Database Schema

### Tables

**users** — Students, staff, and faculty who submit tickets
| Column | Type | Notes |
|--------|------|-------|
| id | SERIAL PK | |
| first_name | VARCHAR(100) | NOT NULL |
| last_name | VARCHAR(100) | NOT NULL |
| email | VARCHAR(150) | UNIQUE, NOT NULL |
| phone | VARCHAR(20) | |
| department | VARCHAR(100) | |
| role | VARCHAR(50) | student / staff / faculty |
| created_at | TIMESTAMP | DEFAULT NOW() |

**technicians** — IT desk workers
| Column | Type | Notes |
|--------|------|-------|
| id | SERIAL PK | |
| first_name | VARCHAR(100) | NOT NULL |
| last_name | VARCHAR(100) | NOT NULL |
| email | VARCHAR(150) | UNIQUE, NOT NULL |
| phone | VARCHAR(20) | |
| specialization | VARCHAR(100) | |
| is_active | BOOLEAN | DEFAULT TRUE |
| created_at | TIMESTAMP | DEFAULT NOW() |

**tickets** — Core ticket records
| Column | Type | Notes |
|--------|------|-------|
| id | SERIAL PK | |
| title | VARCHAR(255) | NOT NULL |
| description | TEXT | NOT NULL |
| status | VARCHAR(50) | Open / In Progress / Resolved / Closed |
| priority | VARCHAR(20) | Low / Medium / High / Critical |
| category | VARCHAR(100) | Network, Hardware, Software, etc. |
| submitter_id | INTEGER FK | → users.id |
| assigned_tech_id | INTEGER FK | → technicians.id |
| ai_suggested_fix | TEXT | Generated by Gemini |
| ai_categorized | BOOLEAN | Was AI used? |
| location | VARCHAR(150) | Campus location |
| created_at | TIMESTAMP | DEFAULT NOW() |
| updated_at | TIMESTAMP | DEFAULT NOW() |
| resolved_at | TIMESTAMP | Set when status = Resolved |

**tags** — Reusable labels
| Column | Type | Notes |
|--------|------|-------|
| id | SERIAL PK | |
| name | VARCHAR(50) | UNIQUE |
| color | VARCHAR(7) | Hex color |

**ticket_tags** — Junction table (many-to-many)
| Column | Type | Notes |
|--------|------|-------|
| ticket_id | INTEGER FK | → tickets.id |
| tag_id | INTEGER FK | → tags.id |

### Relationships
- One **user** submits many **tickets** (one-to-many)
- One **technician** is assigned many **tickets** (one-to-many)
- One **ticket** has many **tags**, one **tag** applies to many **tickets** (many-to-many via `ticket_tags`)

## Setup Instructions

### 1. Clone the repo
```bash
git clone https://github.com/yourusername/gu-it-tickets.git
cd gu-it-tickets
```

### 2. Set up Supabase
1. Go to [supabase.com](https://supabase.com) and create a free project
2. In the SQL Editor, paste and run the contents of `schema.sql`
3. Copy your database connection string from: Settings → Database → Connection string → URI

### 3. Get a Gemini API Key
1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Click "Get API Key" → Create API key (free)

### 4. Configure secrets
Create `.streamlit/secrets.toml`:
```toml
[database]
DB_URL = "postgresql://postgres:[password]@db.[ref].supabase.co:5432/postgres"

[gemini]
GEMINI_API_KEY = "your-gemini-api-key"
```

### 5. Install dependencies and run
```bash
pip install -r requirements.txt
streamlit run Home.py
```

---

## Deployment (Streamlit Community Cloud)

1. Push repo to GitHub (make sure `secrets.toml` is in `.gitignore`)
2. Go to [share.streamlit.io](https://share.streamlit.io)
3. Click "New app" → connect your GitHub repo → set main file to `Home.py`
4. Under "Advanced settings" → add your secrets from `secrets.toml`
5. Deploy!
